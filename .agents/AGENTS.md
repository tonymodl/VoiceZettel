- Писать **самовосстанавливающийся** код (Auto-Reconnect, heartbeat `get_me()`, `try-except` с экспоненциальной задержкой)
- Инициировать миграцию на **Webhooks** для production-среды
- Если Telegram падает — автоматически переподключаться без потери сообщений

## 3. ИНТЕЛЛЕКТУАЛЬНАЯ ПАМЯТЬ И РАБОТА С ХРАНИЛИЩЕМ OBSIDIAN

Ты должен обладать **абсолютной, мгновенной памятью** без задержек на размышления. Твой архив — локальное хранилище Obsidian, содержащее сотни диалогов за долгие годы.

### Архитектура Graph RAG (LightRAG)
> **Игнорируй плоский векторный поиск (ChromaDB)** — он неэффективен и ведёт к потере контекста.

Воспринимай базу данных как единый **Граф Знаний**:
1. **Узлы (Nodes)**: Люди, компании, проекты, обещания, события
2. **Рёбра (Edges)**: Взаимосвязи с атрибутами (тип, дата, сентимент, вес)
3. **Траверсия**: При запросе — рекурсивный обход forward + backlinks до `MAX_DEPTH = 10`

### Паттерн LLM-Wiki (Karpathy)
Три строгих уровня данных:
- **Raw Sources** (`/Raw_v2/`) — иммутабельная директория входящих логов
- **The Wiki** (`/Wiki_v2/`) — структурированный граф Markdown-страниц, управляемый ИИ
- **The Schema** — правила и таксономия сущностей

### Семантическое Разрешение Сущностей
- «Саша», «@alex_dev» в Telegram и «мой брат» в голосовой заметке = **ОДИН узел**
- Склейка по: email, телефон, username, стилистика, контекст упоминаний
- Если неуверен — создай `[[?Unresolved]]` ссылку для ручного подтверждения

Все новые модули разворачиваются **параллельно**, в режиме «только чтение» по отношению к текущим базам и процессам, пока их стабильность не будет доказана.

### Definition of Done (DoD)
- Ни один новый функционал не считается «готовым» без unit-тестов
- Тесты ОБЯЗАНЫ подтвердить, что старые функции работают без задержек
