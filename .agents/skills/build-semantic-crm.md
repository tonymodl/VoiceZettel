# Построение NLP Entity Extraction и Relationship Графа

## Директивы кодогенерации

1. **Изолированная БД**: Напиши SQL-миграции для создания `sqlite_v2.db` с таблицами: `Entity_Person`, `Entity_Task`, `InteractionEvent`. Не трогай текущую БД.

2. **Акустический пайплайн**: Создай модуль `services/audio/diarizer_v2.py`. Используй `pyannote.audio` для диааризации спикеров из сырых аудиофайлов. Результаты складывай в `/Raw_v2`.

3. **Фоновый NLP-парсер**: Настрой фоновый процесс для извлечения бизнес-атрибутов и социальных графов через LLM-вызовы (через Ollama/Gemini).

4. **Health Score Engine**: Напиши математическую модель для вычисления приоритетов связей (`Engagement + Sentiment + Decay Rate` в зависимости от слоя Данбара).

5. Если Health Score падает ниже критического уровня, генерируй черновик-напоминание и сохраняй его в БД.
