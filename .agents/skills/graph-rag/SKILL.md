# Graph RAG — Графовая Память Экзокортекса

## Назначение
Замена неэффективного плоского векторного поиска (ChromaDB k-NN) 
на архитектуру **Graph RAG** с использованием **LightRAG** для мгновенного 
восстановления хронологии, причинно-следственных связей и структуры взаимоотношений.

## Почему ChromaDB неэффективен
- Находит семантически похожие слова, но **теряет хронологическую последовательность**
- Не видит **причинно-следственных связей** между событиями
- Не понимает **структуру взаимоотношений** между людьми
- Изолированные воспоминания не собираются в долгосрочный контекст

## Архитектура LightRAG

### Граф Знаний (Knowledge Graph)
```
[Person: Настя] --жена--> [Person: Антон]
[Person: Настя] --пообещала--> [Event: помочь с отчётом, дата: 2026-04-15]
[Event: помочь с отчётом] --deadline--> [Date: 2026-04-20]
[Person: Костя] --инвестор--> [Project: Dominion]
[Project: Dominion] --milestone--> [Event: запуск MVP, Q2 2026]
```

### Механика траверсии
1. Активный файл (текущая заметка/запрос)
2. Экстракция связанных документов (forward links)
3. Рекурсивная обработка до `MAX_DEPTH = 10`
4. Учёт обратных ссылок (backlinks)
5. Синтез контекста из разрозненных фрагментов

### Гибридный поиск
- **Локальный**: Чтение файлов для точных дат/сумм (высокая точность)
- **Глобальный**: Графовые запросы для синтеза смыслов (широкий контекст)

### Хранилище
```
VoiceZettel/
├── .neural_memory/          # Индекс графа LightRAG
├── Vault/
│   ├── Raw_v2/              # Иммутабельные сырые данные
│   │   ├── Telegram/        # Экспорт переписок
│   │   ├── Voice/           # Голосовые транскрипции
│   │   └── Notes/           # Заметки
│   └── Wiki_v2/             # Структурированный граф (LLM-Wiki)
│       ├── People/          # Карточки людей
│       ├── Projects/        # Карточки проектов
│       ├── Events/          # Хронология событий
│       └── Promises/        # Обещания и обязательства
```

### Интеграция с Obsidian
- Плагин Neural Composer управляет локальным сервером LightRAG
- Сервер запускается/останавливается вместе с Obsidian
- Индекс графа в `.neural_memory/` синхронизирован с заметками
- Нулевая конфигурация

## Миграция с ChromaDB
1. ChromaDB **остаётся работать** на старых данных (Zero Risk)
2. LightRAG строит **параллельный** индекс из Wiki_v2
3. Запросы маршрутизируются: сначала Graph RAG, fallback на ChromaDB
4. Когда Graph RAG доказывает стабильность — ChromaDB деактивируется
